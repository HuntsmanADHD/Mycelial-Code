## ClawedCode Deep Dive

The new **ClawedCode example** demonstrates advanced Mycelial features:

### Network Structure
```
input_messages â†’ {Peer_A, Peer_B, Peer_C}
                    â†“
                 Relay_1
                    â†“
              Consensus_1
                    â†“
             output_messages
```

### Key Features
- **Message routing**: Peers forward messages through relay
- **Consensus**: Nodes vote, threshold triggers decision
- **Resilience**: Works even if one peer fails
- **Realistic**: Message caching, delivery acks, trust scores

### Example Execution Trace
```
Cycle 1:
  INPUT: message(id=1, sender="Alice", content="Hello")
  Peers A, B, C receive and cache
  Each emits vote to Consensus_1

Cycle 2:
  Relay receives 3 copies of message, forwards
  Consensus receives 3 votes
  Threshold (2/3) reached
  Emits: consensus_result(approved=true)

Cycle 3:
  Output receives final decision
  System complete
```
